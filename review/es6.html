杂项 #

Set数据结构 #

  • Set类似于数组，但是它的成员是唯一的，没有重复的值
  • Set本身是一个构造函数，用来生成 Set 数据结构
  • Set函数可以接受一个数组（或者具有 iterable 接口的其他数据结构）作为参数，用来初始化。
// 去除数组的重复成员
[...new Set(array)]

[...new Set('ababbc')].join('')
// "abc"

Set的属性：

  • Set.prototype.constructor：构造函数，默认就是Set函数
  • Set.prototype.size：返回Set成员总数

Set的方法：

  • Set.prototype.add(value)： 添加某个值，返回Set结构本身
  • Set.prototype.delete(value)：删除某个值，返回布尔值表示删除是否成功
  • Set.prototype.has(value)：返回一个布尔值，表示该值是否是Set成员
  • Set.prototype.clear()：清除所有成员，没有返回值
  • Set.prototype.keys()： 返回键名的遍历器
  • Set.prototype.values()： 返回键值的遍历器
  • Set.prototype.entries()： 返回键值对的遍历器
  • Set.prototype.forEach()： 使用回调函数遍历每个成员

Map数据结构 #

Map类似于对象，但是键不限制于字符串，Object 结构提供了字符串—值的对应，Map 结构提供了值—值的对应

const map = new Map();
const val = { p: 'hello' };

map.set(val, 'world');    // 设置元素
map.get(val)  // 'world'  // 获取元素

map.has(val) // true      // 测试是否存在

map.delete(val) // true   // 删除
map.has(val) // false

map.clear(val)            // 清空

Map 也可以接受一个数组作为参数，该数组的成员是一个个表示键值对的数组

const map = new Map([
    ['name', 'yian'],
    ['title', 'Author']
]);
console.log(map);
// Map(2) { 'name' => 'yian', 'title' => 'Author' }

遍历Map数据结构

  • Map.prototype.keys()  ：返回键名的遍历器
  • Map.prototype.values()  ：返回键值的遍历器
  • Map.prototype.entries() ：返回所有成员的遍历器
  • Map.prototype.forEach()  ：遍历 Map 的所有成员

Map数据结构转换 #

Map与数组 #

Map转数组

const myMap = new Map()
  .set(true, 7)
  .set({foo: 3}, ['abc']);

[...myMap] // [ [ true, 7 ], [ { foo: 3 }, [ 'abc' ] ] ]

数组转Map：

const map = new Map(
    [ 
        ['name', 'yian'], 
        ['title', 'Author'] 
    ]
);

console.log(map); // Map(2) { 'name' => 'yian', 'title' => 'Author' }

Map与对象 #

如果Map的键都是字符串可以无损转换，如果键包含其他数据类型则会先转为字符串再作为对象的键名

let obj = Object.create(null);

for (let [ key, value ] of map) { 
    obj[key] = value; 
}

console.log(obj) // { name: 'yian', title: 'Author' }

使用Object.entries()将对象转为Map（ 就相当于先把对象转数组,再将数组转换为Map）

const obj = { 
    "a": 1, 
    "b", 2 
}
const map = new Map(Object.entries(obj)
// [ [ 'a', 1 ], [ 'b', 2 ] ]

Proxy #

Proxy 用于修改某些操作的默认行为，等同于在语言层面做出修改，所以属于一种“元编程”（meta programming），即对编程语言进行编程。

Proxy可以理解成，在目标对象之前架设一层“拦截”，外界对该对象的访问，都必须先通过这层拦截，因此提供了一种机制，可以对外界的访问进行过滤和改写。Proxy 这个词的原意是代理，用在这里表示由它来“代理”某些操作，可以译为“代理器”。

// 实现数组读取负数索引
function proxyArray (target) {
    // 定制拦截行为
    let handler = {
        get(target, propKey, receiver) {
            // 转化为Number类型
            let index = Number(propKey);
            if(index < 0) {
                propKey = String(target.length + index);
            }
            // 每一个Proxy对象的拦截操作,内部都调用对应的Reflect方法，保证原生行为能够正常执行
            return Reflect.get(target, propKey, receiver);
        }
    };
    // 生成Proxy实例
    return new Proxy(target, handler);
}

const targetArray = ['a', 'b', 'c', 'd', 'x', 'y', 'z'];

let arr = proxyArray(targetArray);
console.log(arr[-1]);

Reflect #

Reflect与Proxy一样，也是用来操作对象的。让Object操作都变成函数行为

let obj = { name: 'yian', age: 20 }

Reflect.get(obj, 'name');        // yian

Reflect.set(obj, 'age', 2212);   // { name: 'yian', age: 2212 }

Reflect.has(obj, 'name'); // true

// 等同于delete obj[name]
// delete obj[name]
Reflect.deleteProperty(obj, 'name'); // { age: 2212 }

Promise对象 #

Promise 是异步编程的一种解决方案，单来说就是一个容易，保存着某个未来才会结束的事情。

  • 三种状态: pending(进行中)、fulfilled(已成功)、rejected(已失败)
  • 一旦状态改变就不会再变了，resolved(已定型)
  • Promise无法取消，一旦新建就会立即执行

Promise构造函数接受一个函数作为参数，该函数的两个参数分别是resolve和reject。它们是两个函数，由 JavaScript 引擎提供，不用自己部署

const p = new Promise( function ( resolve, reject ) {
    resolve();
    reject();
})

.then #

  • .then 的第一个参数是一个函数，该函数将在 promise resolved 后运行并接收结果。
  • .then 的第二个参数也是一个函数，该函数将在 promise rejected 后运行并接收 error。
promise.then(
  function(result) { /* handle a successful result */ },
  function(error) { /* handle an error */ }
);

.catch #

用于指定发生错误的函数：

.catch(f) 调用是 .then(null, f) 的完全的模拟，它只是一个简写形式。

finally #

finally方法是无论Promise对象最后状态如何都会执行的操作

promise
.then(result => {···})
.catch(error => {···})
.finally(() => {···});

Iterator和for...of循环 #

Iterator就是一个统一的接口机制，来处理所有不同的数据结构。任何数据结构只要部署Iterator接口就能完成遍历操作。

  • 默认的 Iterator 接口部署在数据结构的Symbol.iterator属性，只要具有这是属性就是可遍历的
  • 原生具备Iterator接口的数据结构为：Array、Map、Set、String、TypedArray、函数的arguments对象、NodeList对象

使用场景：

  • 解构赋值
  • 扩展运算符
  • yield*
  • 任何接收数组为参数的场合

Generator函数 #

异步编程解决方案，以yield表达式定义不同内部状态：

function* hello() {
    yield 'hello';
    yield 'world';
    return 'ending';
}

let a = hello();

yield就是一个状态标志位，每调用一次next()方法，指针移动一次，直到遇到return语句为止

a.next()    // Object { value: "hello", done: false }
a.next()    // Object { value: "world", done: false }
a.next()    // Object { value: "ending", done: true }
a.next()    // Object { value: undefined, done: true }

for...of循环可以自动遍历 Generator 函数运行时生成的Iterator对象，且此时不再需要调用next方法

function* hello() {
    yield '1';
    yield '2';
    return '3';
}

for (let i of hello()){
    console.log(i);
}
// 1 2

async函数 #

async函数就是 Generator 函数的语法糖，使得异步操作更加方便async函数就是将 Generator函数的星号*替换成async，将yield替换成await，仅此而已

let r = async function() {
    let r1 = await readFile('./data/data.json');
    let r2 = await readFile('./data/data.txt');
    console.log('r1: ', r1);
    console.log('r2: ', r2);
    return r2;
}

let rs = r();
console.log('rs: ', rs);
// r1:  [ { "name": "yian" } ]
// r2:  312313124242

rs.then( res => {
    console.log('res: ', res);
})
// res:  312313124242

async函数对于Generator函数的改进

  • 内置执行器
  • 更好的语义
  • 更广的适用性
  • 返回值是Promise

async函数就是将异步转化为同步，等结果出现后再向下执行

function delayTime(ms){
    return new Promise( (resolve, reject) => {
        setTimeout(resolve, ms);
    })
}

async function fn(val, ms) {
    console.log('开始');
    await delayTime(ms); // 等结果出现后再向下执行
    console.log(val);
}

fn('hello', 5000);
// 开始
// ....五秒后
// hello

class #

JS中的class可以看做是构造函数的另一种写法，相当于构造函数的语法糖

  • 静态成员：只能被当前类本身所访问到，需要在对应属性或方法前面加上一个static关键字
  • 动态成员：当前类本身无法访问，只能被所有实例访问
class Person{
    static say(){ 
        console.log('static fn') 
    };
}
Person.say() // static fn

let user = new Person;
user.say();  // TypeError: user.say is not a function

注意：

  • 严格模式：类和模块内部默认就是严格模式
  • 不存在变量提升
  • name属性
  • Generator方法
  • this指向
  • super关键字（参见：类的继承）
