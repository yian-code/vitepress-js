代码规范 #

这是风格偏好，不是宗教教条。当然，大公司有自己的代码规范手册，不遵守规范的叫粑粑山

  • 不要省略分号

JavaScript将换行符理解为隐式分号，也被称为自动分号插入。但是JavaScript存在无法确定是否真的需要自动插入分号的情况，下面就是一个错误的案例：

console.log('hello')

[1, 2].push('3');
/* hello 
 * Uncaught TypeError: console.log(...) is undefined 
 */
  • 注释

注释是代码的重要组成部分，Ctrl + / 单行注释，Ctrl + Shift + A 多行注释

  • var已是过去式

现代脚本中一般不在使用var，了解var和let的区别，以避免奇怪的错误，参见变量一节

  • 所有函数都应该设置为常量

  • 使用数组成员对变量赋值时优先使用解构赋值

const arr = [1, 2, 3, 4];

// bad 
const a = arr[0];
const b = arr[1];

// good 
const [a, b] = arr;

  • 函数参数为对象的成员时，优先使用解构赋值

  • 函数返回多个返回值时，优先使用对象的解构赋值而不是数组的解构赋值，这样便于添加返回值以及改变返回值的顺序

  • 单行对象最后一个成员不以逗号结尾，多行对象最后一个成员以逗号结尾

  • 对象尽量静态化，一旦创建就不得随意添加属性，添加属性使用Object.assing()

  • 对象属性是动态的则使用属性表达式定义

  • 对象属性和方法尽量使用简洁表达方式

  • 使用扩展运算符拷贝数组

const arr = [1, 2, 3, 4];
const arrCopy = [...arr];

arr[2] = 100;

console.log(arrCopy); // [ 1, 2, 3, 4 ]
console.log(arr) // [ 1, 2, 100, 4 ]
  • 使用默认值语法设置参数默认值
  • 总是用 Class取代需要 prototype 的操作。因为 Class 的写法更简洁，更易于理解
  • 优先使用箭头函数，但是要知道有的地方是不能使用箭头函数的!
  • 使用rest参数代替arguments对象
  • 使用模板字符串代替连接字符串
  • 字符串使用单引号，不使用双引号
  • 优先使用===和!==，而不是== !==
  • 生产环境记得清除console.log()
  • ....
