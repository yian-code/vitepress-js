变量：const、var、let #

const #

const用来创建常量，可以清楚的告诉别人这是一个不能改变的常数，使用const关键字创建的常量不能被修改，如果你试图修改它则会发现报错

const a = '1';
a = '2';      // 未捕获的类型错误:对const 'A'的赋值无效

大写的常数 #

一个普遍的做法是将常量用作别名，以便记住那些在执行之前就已知的难以记住的值。

使用大写字母和下划线来命名这些常量。

const COLOR_RED = '#F00';

// 使用时
let color = COLOR_RED;
  • 一目了然，知道这是个常数
  • 更容易记忆

let与var #

let 与 var 相似，大部分情况下我们使用它们俩相互替代都是可以的，但是实际上var却很不同，了解将旧脚本从 var 迁移到 let 时的区别，以避免奇怪的错误，是很重要的。

var没有块级作用域 #

  • var声明的变量没有块级作用域概念，只有函数作用域和全局作用域
  • let声明的变量有块级作用域
{
    let a = 1;
    var b = 2;
}

console.log('a:', a); //Uncaught ReferenceError: a is not defined
console.log('b:', b);

var 可以穿透 if、for 和其它代码块。这是因为在早期的 JavaScript 中，块没有词法环境，而 var 就是这个时期的代表之一。

var允许重新声明 #

当使用let对变量进行两次声明时会报错

let a = 1;
let a = 2; // SyntaxError: 'user' has already been declared

而var允许重复的对同一个变量进行声明

var a = 1;
var a = 2; // a = 2

var允许声明前调用 #

当函数开始的时候，就会处理 var 声明（脚本启动对应全局变量）。

var 声明的变量会在函数开头被定义，与它在代码中定义的位置无关（这里不考虑定义在嵌套函数中的情况）。

phrase = "Hello";
console.log(phrase); // Hello
var phrase;

以上代码相当于

var phrase;
phrase = "Hello";
console.log(phrase); // Hello

这种行为称为“提升”（变量提升），因为所有的 var 都被“提升”到了函数的顶部，而let不存在变量提升

声明会被提升，但是赋值不会。

console.log('a:', a); // undefined
var a = 1;

var a = 1 这行代码包含两个行为：

  1. 使用 var 声明变量
  2. 使用 = 给变量赋值

所以，以上代码相当于

var a;  // undefined
console.log('a:', a);
a = 1;

所有的 var 声明都是在函数开头处理的，我们可以在任何地方引用它们。但是在它们被赋值之前都是 undefined。
