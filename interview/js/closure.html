作用域和闭包、this #

作用域 #

  • 全局作用域
  • 函数作用域
  • 块级作用域
if(true) {
    let x = 200;
}
console.log(x);  // 报错

浏览器的全局执行环境为window对象，所有的全局变量和函数都作为window对象的属性和方法创建

在node环境下， 全局执行环境为global对象

自由变量 #

  • 一个变量在当前作用域没有定义，但是被使用了
  • 向上级作用域，一层一层依次查找，直到找到为止
  • 如果到全局作用域都没找到，则报错xx is not defined

闭包 #

闭包是作用域应用的特殊情况，闭包一般有两种表现：

  • 函数作为参数被传递
function print(fn) {
    const a = 200;
    fn();
}

const a = 100;
function fn() {
    console.log('a: ', a);
}

print(fn);   // a: 100
  • 函数作为返回值被返回
function create() {
    const a = 100;
    return function () {
        console.log('a: ', a);
    }
}
const fn = create();
const a = 200;
fn();       // a: 100

闭包总结：自由变量的查找，是在函数定义的地方向上一级作用域查找，而不是在执行的地方！！

this #

JS中this的应用场景：

  • 作为普通函数调用
  • 使用call、apply、bind
  • 作为对象方法被调用
  • 在class方法中调用
  • 箭头函数

this取什么值是在函数执行的时候确定的，不是在函数定义的时候确定的。

function fn1() {
    console.log('this:', this);
}
fn1();            // this: Window

fn1.call('100');  // this: String { "100" }

const fn2 = fn1.bind('200'); 
fn2(); 			  // this: String { "200" }

call直接调用执行，bind要返回给一个新的函数，然后重新执行新的函数执行。

call、apply、bind #

  • call apply bind三者都可以实现函数中this指向的修改。

  • bind不会直接调用函数执行，bind要返回给一个新的函数，然后重新执行新的函数执行（看上一个例子）。

  • apply方法有且仅有两个参数，第二个参数是一个数组类型的值，数组中的每一个数组项表示调用该函数时传递的实参 var sum2 = add.apply(this,[20,30,40])。

var arr = [123,2,234,234,23,5,3,6,54,7,56,756,7,567];

var max1 = Math.max.apply(this,arr); // 使用apply将arr作为实参传入
console.log(max1); // 756

apply一般在当我们需要将一个数组中的每一项作为一个函数的实参进行一一传递时,那么我们就可以使用apply进行函数的调用

class中的this #

class Person {
    constructor(name) {
        this.name = name;
        this.age = 20;
    }
    sayHi() {
        console.log(this);
    }
}

const yian = new Person('yian');
yian.sayHi();  // 当前对象：Object { name: "yian", age: 20 }

箭头函数没有自己的this #

箭头函数没有 this。如果访问this，则会从外部获取。

const yian = {
    name: 'yian',
    sayHi(){ 
        console.log(this)  // this 即当前对象
    },
    wait() {
        setTimeout( function() {
            console.log(this);  // this === window
        })
    }
}

当我们将setTimeout中的函数写成箭头函数形式，即：

wait() {
    setTimeout( () => console.log(this) ) // this 即当前对象
}

任务1：理解this #

在 JavaScript 中，this 关键字与其他大多数编程语言中的不同。JavaScript 中的 this 可以用于任何函数，即使它不是对象的方法。

  • 在全局作用域下this表示的是window对象
  • 在函数中this表示该函数的调用者
  • 在事件中this表示触发该事件的标签
  • 在定时器的回调函数中this表示的是window
  • 在构造函数中this表示构造函数的实例
  • 在原型对象中this表示该原型的实例
  • 在箭头函数中没有自己的this指向, 上级作用域的this指向谁 ,箭头函数中的this就指向谁

任务2：手写bind函数 #

手写bind函数，函数名myBind：

function fn1(a, b, c) {
    console.log('this: ', this);
    console.log(a, b, c);
    return 'this is fn1';
}

const fn2 = fn1.bind({x: 100}, 10, 20, 30);
fn2();

/* 
this: Object { x: 100 }
10 20 30
"this is fn1" 
*/
Details
// 模拟bind
Function.prototype.myBind = function () {
    // 将参数拆解为数组
    const args = Array.prototype.slice.call(arguments);

    // 获取 this （数组第一项）
    const t = args.shift(); // {x: 100}

    // fn1.bind(...) 中的fn1
    const self = this;

    // bind返回一个函数，这里我们也返回一个函数
    return function () {
        return self.apply(t, args);
    }
}

运行：

const fn2 = fn1.myBind({x: 100}, 10, 20, 30);
fn2();

/* 
this: Object { x: 100 }
10 20 30
"this is fn1" 
*/

闭包在实际开发中的应用 #

  1. 隐藏数据
function createCache() {
    const data = {}; // 闭包中的数据，被隐藏，不被外界访问 
    return {
        set: function (key, val) {
            data[key] = val;
        },
        get: function (key) {
            return data[key];
        }
    }
}

const c = createCache();
c.set('yi', 100);

console.log('yi: ', c.get('yi')); // yi: 100

data.b = 200  // Uncaught ReferenceError: data is not defined

任务3：点谁打印谁 #

创建10个a标签，点谁打印谁的序号

以下是错误的实现案例：无论点击哪个都是打印 10

let i, a;
for (i = 0; i < 10; i++) {
    a = document.createElement('a');
    a.innerHTML = i + '<br>';

    a.addEventListener('click', function(e) {
        e.preventDefault();
        console.log(i);
    })
    document.body.appendChild(a);   
}

原因：i 是全局变量，创建的click点击事件在点击时才会执行，i 早已经变成 10 了，所以每次打印的都是10

Details
let a;
for (let i = 0; i < 10; i++) {
    a = document.createElement('a');
    a.innerHTML = i + '<br>';

    a.addEventListener('click', function(e) {
        e.preventDefault();
        console.log(i);
    })
    document.body.appendChild(a);   
}

将全局作用域的i修改为块级作用域

补充 #

任务3配合变量理解，作用域的问题

let允许你声明一个作用域被限制在块作用域中的变量、语句或者表达式，暂时性死区

for(let a = 1; a < 5; a++){
    setTimeout(() => { console.log('a:', a); }, 0);
    console.log(a);
}
// a: 1
// a: 2
// a: 3
// a: 4

var声明的变量作用域是全局或者整个函数块的.不存在死区

for(var a = 1; a < 5; a++){
    setTimeout(() => { console.log('a:', a); }, 0);
    console.log(a);
}
// a: 5
// a: 5
// a: 5
// a: 5
