存储 #

cookie #

Cookie 是直接存储在浏览器中的一小串数据。它们是 HTTP协议的一部分。

Cookie 通常是由 Web 服务器使用响应 Set-Cookie HTTP-header 设置的。然后浏览器使用 Cookie HTTP-header 将它们自动添加到（几乎）每个对相同域的请求中。

最常见的用处就是身份验证：我们可以直接使用document.cookie从浏览器中访问cookie。

写入cookie：

document.cookie = "user=John"; // 只会更新名称为 user 的 cookie
alert(document.cookie); // 展示所有 cookie

document.cookie= 操作不是重写整所有 cookie。它只设置代码中提到的 cookie user。

  • 存储大小，最大4KB
  • http请求时需要发送到服务端，增加了请求数据量
  • 只能用document.cookie=修改，不好用

HTML5存储 #

Web 存储对象 localStorage 和 sessionStorage 允许我们在浏览器上保存键/值对。

  • HTML5专门为存储设计的，最大可存5M
  • API简单易用，setItem getItem
  • 不会随http请求被发送出去

两个存储对象都提供相同的方法和属性：

  • setItem(key, value) —— 存储键/值对。
  • getItem(key) —— 按照键获取值。
  • removeItem(key) —— 删除键及其对应的值。
  • clear() —— 删除所有数据。
  • key(index) —— 获取该索引下的键名。
  • length—— 存储的内容的长度。

它就像一个 Map 集合（setItem/getItem/removeItem），但也允许通过 key(index) 来按索引访问。

localStorage 和 sessionStorage区别 #

  • localStorage
    • 在同源的所有标签页和窗口之间共享数据。(只需要在同一个源（域/端口/协议），URL 路径可以不同，在所有同源的窗口之间，localStorage 数据可以共享)
    • 数据不会过期。它在浏览器重启甚至系统重启后仍然存在。
  • sessionStorage（不常用）
    • sessionStorage 的数据只存在于当前浏览器标签页。
    • 具有相同页面的另一个标签页中将会有不同的存储。
    • 但是，它在同一标签页下的 iframe 之间是共享的（假如它们来自相同的源）。
    • 数据在页面刷新后仍然保留，但在关闭/重新打开浏览器标签页后不会被保留。
