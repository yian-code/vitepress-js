事件 #

DOM0级事件 #

DOM 常用事件，基本语法：

const btn = document.getElementById('btn1');

btn.onclick = function (params) { ... }
// btn['onclick'] = function (params) { ... }  同上

当绑定的函数有参数时：标签.onclick = function(){函数调用}

function add(x, y) { console.log(x + y) };

btn.onclick = function() {
    add(1, 2);
}

如果需要移出事件，则将其赋空即可：

btn.onclick = null;

addEventListener #

addEventListener属于DOM2级事件（函数式），它与DOM0级事件（赋值式）的区别在于：

  • DOM0级事件同时绑定多个相同事件时，后绑定的事件会将先绑定的事件覆盖掉，DOM2级事件可以同时绑定相同事件，不会被覆盖。
  • DOM0级事件无法控制事件响应阶段（例如冒泡），DOM2级事件可以设置附加可选对象来控制事件流的响应阶段。
const btn = document.getElementById('btn1');
btn.addEventListener('click', event => {
    event.target
    console.log('btn1 is clicked!');
})

冒泡 #

冒泡原理：当一个事件发生在一个元素上，它会首先运行在该元素上的处理程序，然后运行其父元素上的处理程序，然后一直向上到其他祖先上的处理程序。

阻止冒泡：

event.stopPropagation(); // 阻止冒泡

阻止浏览器默认事件：

event.preventDefault();  // 阻止默认行为

removeEventListener #

要移除处理程序，我们需要传入与分配的函数完全相同的函数。

这样是不起作用的

elem.addEventListener( "click" , () => alert('Thanks!'));
// ....无法移出
elem.removeEventListener( "click", () => alert('Thanks!'));

正确的做法：

function handler() {
  alert( 'Thanks!' );
}

input.addEventListener("click", handler);
// ....
input.removeEventListener("click", handler);

如果我们不将函数存储在一个变量中，那么我们就无法移除它。由 addEventListener 分配的处理程序将无法被“读回”。

事件对象（event） #

当事件发生时，浏览器会创建一个 event 对象，将详细信息放入其中，并将其作为参数传递给处理程序。

const btn = document.getElementById('btn1');
btn.addEventListener('click', event => {
    console.log(event.target)  // 获取触发事件的元素
})
  • event.type：事件类型
  • event.currentTarget：处理事件的元素。这与 this 相同，除非处理程序是一个箭头函数，或者它的 this 被绑定到了其他东西上，之后我们就可以从 event.currentTarget 获取元素了。
  • event.clientX / event.clientY：指针事件（pointer event）的指针的窗口相对坐标。

事件代理 #

事件代理也叫事件委托，是利用了事件的冒泡机制，来处理有很多子元素需要绑定事件的一种解决方式。

/* 
<li> 
    <button class="btn1">按钮一</button> 
    <button class="btn2">按钮二</button> 
</li> 
*/
var li = document.querySelector('li');

li.onclick = function(event){
    var b = event.target; // 在哪里点击的？ 
    if(b.className == "btn1"){
        console.log("按钮一点击")
    }else if(b.className == "btn2"){
        console.log("按钮二点击")
    }else{
        console.log("点击了大盒子")
    }
}

事件代理的优点：

  • 简化初始化并节省内存：无需添加许多处理程序。
  • 更少的代码：添加或移除元素时，无需添加/移除处理程序。
  • DOM 修改 ：我们可以使用 innerHTML 等，来批量添加/移除元素。
  • 但是，不要滥用事件代理

局限性：

  • 事件必须冒泡，而有些事件不会冒泡。此外，低级别的处理程序不应该使用 event.stopPropagation()。

几乎所有事件都会冒泡，也有例外。例如，focus 事件不会冒泡

  • 其次，委托可能会增加 CPU 负载，因为容器级别的处理程序会对容器中任意位置的事件做出反应，而不管我们是否对该事件感兴趣。通常负载可忽略不计，所以不必考虑。

任务：手写通用事件绑定函数 #

// 不考虑事件代理的通用事件绑定函数
function bindEvent(elem,type,fn) {
    elem.addEventListener(type,fn)
}

const div1 = document.getElementById("div1")
bindEvent(div1,'click',event => {
    console.log("点击")
    console.log(event.target)
})

⭐考虑事件代理的通用事件绑定函数（这个需要好好理解消化）

function bindEvent(elem, type, selector, fn) {
    if(fn == null) { // 三个参数与四个参数的判断处理
        fn = selector;
        selector = null;
    }
    elem.addEventListener(type, event => {
        const target = event.target;
        if(selector) {
            // 代理绑定
            if(target.matches(selector)) {
                fn.call(target, event);
            }
        } else {
            // 普通绑定
            fn.call(target.event);
        }
    })
}

对target.matches(selector)的说明

Element.matches()：如果元素被指定的选择器字符串选择，Element.matches() 方法返回 true; 否则返回 false。
