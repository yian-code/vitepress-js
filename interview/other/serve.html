运行环境（性能与安全） #

运行环境即浏览器（server端有nodejs）

下载网页代码，渲染页面都会执行很多JS，就需要考虑性能优化，保证代码在浏览器中稳定高效的运行

页面加载和渲染过程 #

参考面试题：从输入url到渲染出页面的整个过程

  1. 资源的形式
  • html代码
  • 媒体文件
  • javascript、css
  1. 加载过程
  • DNS解析：域名 -> IP地址
  • 浏览器根据IP地址向服务器发送http请求
  • 服务器根据http请求，返回给浏览器
  1. 渲染过程
  • 根据HTML生成DOM Tree
  • 根据CSS生成CSSOM
  • 将DOM Tree和CSSOM整合生成Render Tree
  • 根据Render Tree渲染页面
  • 遇到<script>则暂停渲染，优先加载JS代码，完成后继续（JS和DOM渲染共用一个线程，因为JS可能会更改DOM结构）

性能优化 #

面试题：防抖和节流，图片懒加载

性能优化原则 #

  • 多使用内存，缓存或其他方法
  • 减少CPU计算量，减少网络加载耗时

性能优化方法 #

让加载更快：

  • 减少资源体积：压缩代码
  • 减少访问次数：合并代码，SSR服务端渲染，缓存
  • 使用更快的网络：CDN

让渲染更快：

  • CSS放在head，JS放在body最下面
  • 尽早执行JS，用DOMContentLoaded触发
  • 懒加载（图片懒加载，上滑加载更多）
  • 对DOM查询进行缓存
  • 频繁的DOM操作，合并到一起插入DOM结构
  • 节流（throttle）防抖（debounce）

示例 #

缓存 #

  • 静态资源加hash后缀，根据文件内容计算hash
  • 文件内容不变则hash不变，则url不变
  • url和文件不变，则会自动触发http缓存机制，返回304

SSR #

  • 服务端渲染：将网页和数据一起加载，一起渲染
  • 非SSR（前后端分离）：先加载网页，再加载数据，再渲染页面

图片预加载和懒加载 #

// 预加载
const imgUrl = ["./imgs/1.png","./imgs/2.png","./imgs/3.png"];
let img = new Image();
img.src = imgUrl[0];
img.onload = function(){	
    // 图片加载完再执行的操作
}
// 懒加载
window.onscroll = function(){
    //获取浏览器滚动的距离
    let sTop = document.documentElement.scrollTop || document.body.scrollTop;
    //获取整个页面的高度
    let bodyHeight = body.offsetHeight
    //获取浏览器视口的高度
    let windowHeight = window.innerHeight;
    // 判断滚动是否到达底部,如果到达底部则加载更多图片
    if(sTop >= bodyHeight - windowHeight - 50){
        // 网络请求,加载其他图片
    }
}

防抖（debounce） #

防止频繁的触发执行某一任务：

const ipt = document.getElementById('input1');
// let timer = null;
// ipt.addEventListener('keyup', function () {
//     if(timer) {
//         clearTimeout(timer);
//     }
//     timer = setTimeout(() => {
//         // 模拟触发change事件
//         console.log(ipt.value);

//         // 清空定时器
//         timer = null;
//     }, 300);
// })

// 函数封装
function debounce(fn, delay = 500) {
    let timer = null; // timer在闭包中
    return function() {
        if(timer) {
            clearTimeout(timer);
        }
        timer = setTimeout(() => {
            // debounce有可能会传一些参数，这里接收this还有参数
            fn.apply(this, arguments); 
            timer = null;
        }, delay);
    }
}
// 使用 （debounce调用就不能用箭头函数了，否则没this）
ipt.addEventListener('keyup', debounce( function () {
    // 模拟触发change事件
    console.log(ipt.value);
}))

最常见的就是页面滚动条监听：

let timer;
window.onscroll = function () {
    clearTimeout(timer);
    timer = setTimeout(function () {
        //滚动条位置
        let scrollTop = document.body.scrollTop || document.documentElement.scrollTop;
        console.log('滚动条位置：' + scrollTop);
    }, 200)
}

节流（throttle） #

当持续触发事件时，保证一定时间段内只调用一次事件处理函数。

  • 图片懒加载
  • ajax数据请求加载
  • 拖拽一个元素，需要随时拿到元素的位置，如果直接使用drag事件则会频繁触发，很容易卡顿。节流：无论拖拽多快，都会隔100ms触发一次
<div id="div1" draggable="true">可拖拽</div>

const div1 = document.getElementById('div1');
div1.addEventListener('drag', function(e) {
    console.log(e.offsetX, e.offsetY);
})

使用节流进行优化

const div1 = document.getElementById('div1');

let timer = null;
div1.addEventListener('drag', function(e) {
    if(timer) {
        return
    }
    timer = setTimeout( () => {
        console.log(e.offsetX, e.offsetY);
        timer = null;
    }, 100)
})

对节流进行函数封装

function throttle(fn, delay = 100) {
    let timer = null;

    return function() {
        if(timer){ 
            return 
        }
        timer = setTimeout( () => {
            fn.apply(this, arguments);
            timer = null;
        },delay)
    }
}

div1.addEventListener('drag', throttle( function() {
     console.log(e.offsetX, e.offsetY);
}))

安全 #

  • XSRF跨网站请求伪造攻击，参考存储

预防XSRF：

  • 使用POST接口

  • 增加验证，例如密码，短信验证码，指纹等

  • XSS跨站请求攻击

预防XSS攻击：

  • 替换特殊字符，如<变为&lt;，>变为&gt;
  • <script>变为&lt;script&gt;，直接显示而不会作为脚本执行
  • 前端要替换，后端也要替换
